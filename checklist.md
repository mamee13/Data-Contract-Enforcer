# Week 7 Data Contract Enforcer — 5/5 Checklist

This checklist is distilled from the project brief and practitioner manual. It is scoped to what must be **delivered** to score **5/5** on every rubric metric. Use it as a build and submission gate.

---

## ContractGenerator — 5/5 Deliverables

1. `contracts/generator.py` runs via CLI without errors.
2. Generates Bitol-compatible YAML contracts at minimum:
3. `generated_contracts/week3-document-refinery-extractions.yaml`
4. `generated_contracts/week5-event-records.yaml`
5. Each contract includes **structural + statistical** clauses (not just schema).
6. LLM annotations are present for ambiguous columns.
7. Lineage context injected (`lineage.downstream` populated).
8. dbt schema.yml output exists for each contract:
9. `generated_contracts/week3-document-refinery-extractions_dbt.yml`
10. `generated_contracts/week5-event-records_dbt.yml`
11. Contract quality proof: `DOMAIN_NOTES.md` shows >70% clauses survive manual review, with the measured fraction documented.

---

## ValidationRunner — 5/5 Deliverables

1. `contracts/runner.py` runs via CLI and **always** produces JSON report (even on errors).
2. Report matches required schema exactly (PASS/FAIL/WARN/ERROR per clause).
3. Partial failures handled: missing columns => `ERROR`, runner continues.
4. Structural checks + statistical checks + drift detection implemented.
5. Detects injected violation (e.g., confidence scaled 0–100).
6. Drift check catches 0.0–1.0 → 0–100 scale change even if type passes.
7. At least one **real** validation report in `validation_reports/` from your own data.

---

## ViolationAttributor — 5/5 Deliverables

1. `contracts/attributor.py` runs and produces a blame chain.
2. Uses lineage traversal from Week 4 graph.
3. Integrates `git log` / `git blame` and outputs ranked candidates with confidence.
4. Writes to `violation_log/violations.jsonl` with required schema.
5. Blast radius included (from lineage downstream info).
6. At least one **real violation** traced to actual commit in your git history.
7. Never returns fewer than 1 or more than 5 blame candidates.

---

## SchemaEvolutionAnalyzer — 5/5 Deliverables

1. `contracts/schema_analyzer.py` runs and diffs snapshots.
2. ContractGenerator writes timestamped snapshots to `schema_snapshots/` on every run.
3. Analyzer classifies changes using the taxonomy.
4. Generates migration impact report with:
5. exact diff
6. compatibility verdict
7. blast radius
8. migration checklist
9. rollback plan
10. CLI can show a **breaking change** classification.

---

## AI Contract Extensions — 5/5 Deliverables

1. `contracts/ai_extensions.py` implements all 3 extensions:
2. embedding drift
3. prompt input schema validation
4. LLM output schema enforcement
5. Embedding drift runs on **real** extracted_facts text.
6. Prompt input violations quarantined to `outputs/quarantine/`.
7. LLM output schema violation rate tracked and written.
8. Rising violation rate triggers WARN and appears in violation log.
9. Demo shows a real drift or a real baseline with explicit note.
10. Results appear in validation reports or AI metrics JSON.

---

## Enforcer Report — 5/5 Deliverables

1. Auto-generated by `contracts/report_generator.py`.
2. Outputs `enforcer_report/report_data.json` with real numbers.
3. Report PDF contains all 5 required sections:
4. Data Health Score + narrative
5. Violations this week (severity counts + top 3)
6. Schema changes detected
7. AI system risk assessment
8. Recommended actions (concrete, file-level)
9. Non-engineer can identify correct action without explanation.
10. Report derived from `validation_reports/` + `violation_log/` (not hardcoded).

---

## DOMAIN_NOTES.md — 5/5 Deliverables

1. Minimum 800 words.
2. All 5 questions answered with evidence from your Week 1–5 outputs.
3. Includes **actual output** of the confidence distribution script.
4. Bitol YAML examples are syntactically valid.
5. Confidence scale change traced end-to-end (Week 3 → Week 4 → blame chain).
6. Answers show predictive failure reasoning (not just definitions).

---

## Required Repo Structure (Non‑negotiable)

1. `contracts/` contains all 6 entry-point scripts.
2. `generated_contracts/` contains YAML + dbt YAML.
3. `validation_reports/` contains real run outputs.
4. `violation_log/violations.jsonl` contains ≥ 3 violations.
5. `schema_snapshots/` contains ≥ 2 snapshots per contract.
6. `enforcer_report/` contains generated report.
7. `outputs/` includes week1–week5 + traces JSONL.
8. `README.md` gives step-by-step run commands + expected outputs.
